## Why

NPC dialogue generated by the AI service is currently text-only. Adding TTS via Pocket TTS makes responses audible, increasing immersion significantly. The mic_python service already runs locally and has sounddevice available, making it the natural host for TTS playback with no new processes required.

## What Changes

- **mic_python gains a `--tts` flag** that loads the Pocket TTS model and a voice cache at startup (off by default; model is large)
- **Voice profiles** (`.safetensors` files) are placed in `mic_python/voices/` and eagerly loaded at startup; named by faction raw engine key (e.g. `dolg_1`, `bandit_1`)
- **`export_voices.bat` + `export_voices.py`** bake `.wav` reference files into `.safetensors` via the venv, run once when new voices are added
- **Voice ID assignment** in Lua: each character is assigned a `voice_id` string (e.g. `"dolg_3"`) on first encounter using the raw engine faction name, persisted in save data identically to backstory
- **Shared task queue** in mic_python: STT (mic) and TTS tasks share a single IDLE/BUSY state machine to prevent simultaneous recording and playback
- **Lua TTS queue**: `dialogue.display` is intercepted by Lua; if TTS is enabled, the text is queued and dispatched to mic_python one-at-a-time via `tts.speak`; HUD text is shown only after `tts.started` is received
- **Three new ZMQ topics**: `tts.speak` (Lua → mic_python), `tts.started` (mic_python → Lua), `tts.done` (mic_python → Lua)
- **`launch_mic.bat`** gets "with TTS" variants for each provider option
- **`docs/zmq-api.yaml`** documents the three new topics

## Capabilities

### New Capabilities

- `mic-tts-engine`: TTS generation pipeline in mic_python — `--tts` flag, startup voice cache loading, streaming audio playback via sounddevice, `tts.speak`/`tts.started`/`tts.done` ZMQ handler/publishers, shared STT+TTS task queue with state machine
- `voice-profile-store`: Lua-side per-character voice_id assignment using raw engine faction name, random selection from `voice_data.lua` pool, persistence in save data, serialisation into ZMQ character payloads

### Modified Capabilities

- `mic-zmq-transport`: Extended with three new ZMQ topics (`tts.speak`, `tts.started`, `tts.done`) and the shared task queue replacing the current single-purpose mic loop
