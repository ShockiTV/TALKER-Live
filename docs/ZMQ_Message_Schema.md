# ZMQ Message Schema

This document describes the message format and topics used for communication between the STALKER Anomaly game (Lua) and the Python service via ZeroMQ.

## Transport Protocol

- **Pattern**: PUB/SUB (Publisher/Subscriber)
- **Direction**: Lua (PUB) → Python (SUB)
- **Endpoint**: `tcp://127.0.0.1:5555` (configurable via MCM)
- **Message Format**: `<topic> <json-payload>`

## Topics

### `game.event`

Published when game events occur (death, dialogue, artifacts, etc.).

**Payload Structure:**
```json
{
  "event": {
    "type": "DEATH|DIALOGUE|CALLOUT|TAUNT|ARTIFACT|ANOMALY|MAP_TRANSITION|EMISSION|INJURY|SLEEP|TASK|WEAPON_JAM|RELOAD|IDLE|ACTION",
    "context": {
      // Event-specific fields, see Context Types below
    },
    "game_time_ms": 123456789,
    "world_context": "In Cordon at morning during clear weather, next to a lit campfire.",
    "witnesses": [
      // Array of Character objects
    ],
    "flags": {
      "is_silent": false,
      "is_idle": false,
      "is_compressed": false,
      "is_synthetic": false
    }
  },
  "is_important": true
}
```

### `player.dialogue`

Published when the player types in the chatbox.

**Payload Structure:**
```json
{
  "text": "Hello, how are you?",
  "context": {
    "speaker": { /* Character object */ }
  }
}
```

### `player.whisper`

Published when the player whispers (companion-only communication).

**Payload Structure:**
```json
{
  "text": "Follow me quietly",
  "target": { /* Character object or null */ }
}
```

### `config.update`

Published when an MCM setting changes.

**Payload Structure:**
```json
{
  "key": "setting_name",
  "value": "new_value"
}
```

### `config.sync`

Published on game load with full MCM configuration.

**Payload Structure:**
```json
{
  "model_method": 1,
  "model_name": "gpt-4",
  "fast_model_name": "gpt-3.5-turbo",
  "temperature": 0.7,
  "max_tokens": 150,
  "zmq_enabled": true,
  "zmq_port": 5555,
  "zmq_heartbeat_interval": 5,
  "language": "English",
  "player_speaks": true,
  "action_descriptions": false
  // ... additional MCM settings
}
```

### `system.heartbeat`

Published periodically to indicate the game is running.

**Payload Structure:**
```json
{
  "game_time_ms": 123456789,
  "timestamp": "2026-01-29T12:00:00"
}
```

## Data Types

### Character Object

Represents an NPC or player character.

```json
{
  "game_id": "12345",
  "name": "Wolf",
  "faction": "stalker",
  "experience": "Expert",
  "reputation": "Great",
  "personality": "friendly, cautious",
  "backstory": "Veteran of many Zone expeditions",
  "weapon": "AK-74",
  "visual_faction": null
}
```

**Fields:**
- `game_id`: Unique ID from the game engine (string)
- `name`: Display name
- `faction`: Faction identifier (stalker, Duty, Freedom, Bandit, etc.)
- `experience`: Rank (Novice, Trainee, Experienced, Professional, Veteran, Expert, Master, Legend)
- `reputation`: Reputation tier (Terrible, Bad, Neutral, Good, Great, Excellent)
- `personality`: Personality traits (comma-separated)
- `backstory`: Character backstory text
- `weapon`: Currently equipped weapon
- `visual_faction`: Disguise faction if disguised, null otherwise

### Event Context Types

Context fields vary by event type:

**DEATH:**
```json
{
  "victim": { /* Character */ },
  "killer": { /* Character */ }
}
```

**DIALOGUE:**
```json
{
  "speaker": { /* Character */ },
  "text": "Hello there!"
}
```

**CALLOUT:**
```json
{
  "spotter": { /* Character */ },
  "target": { /* Character */ }
}
```

**ARTIFACT:**
```json
{
  "actor": { /* Character */ },
  "action": "pickup|use|equip",
  "item_name": "Moonlight"
}
```

**MAP_TRANSITION:**
```json
{
  "actor": { /* Character */ },
  "from_location": "Cordon",
  "to_location": "Garbage",
  "companions": [ /* Array of Characters */ ]
}
```

**INJURY:**
```json
{
  "victim": { /* Character */ },
  "health_percent": 15
}
```

**TASK:**
```json
{
  "actor": { /* Character */ },
  "action": "completed|failed|started",
  "task_name": "Find the documents"
}
```

### Event Flags

```json
{
  "is_silent": false,      // Event stored but no dialogue generated
  "is_idle": false,        // Idle conversation trigger
  "is_compressed": false,  // Synthetic compressed memory event
  "is_synthetic": false    // Generated by the system, not game
}
```

## Notes

### Empty Tables

Lua serializes empty tables `{}` as JSON arrays `[]`. The Python service handles this gracefully using a `get_flags()` helper method that returns an empty dict regardless of whether flags is `[]` or `{}`.

### Character ID "0"

The player character always has `game_id = "0"`.

### Fire-and-Forget

All Lua→Python messages are fire-and-forget. The game does not wait for or expect responses. Message loss is acceptable in Phase 1.
